# Obsidian 整理规则

## 核心原则

- 只用中文分类，越简单越好。
- 新内容不再使用 `01`、`02`、`10`、`20`、`99` 这类数字前缀。
- 旧数字目录暂时保留为历史资料，不新增内容，不急着删除。
- 所有大项目、文案、Agent 内容都要能从首页或总览页找到。

## 新分类

| 分类 | 放什么 |
|---|---|
| 项目 | 商业项目、产品项目、网站、工具、长期实验 |
| 文案 | 公众号、短视频、朋友圈、社群、官网、销售话术 |
| 智能体 | Grace、Mia、Atlas、Lisa、Hygeia 的规则、资料、交接 |
| 资料 | 文章、报告、研究、视频知识、工具观察 |
| 素材 | 图片、配色、设计参考、附件 |
| 系统 | Obsidian 规则、同步规范、索引说明 |

## 大项目同步规则

每个大项目默认建立：

```text
项目/项目名/
  项目总览.md
  当前进展.md
  决策记录.md
  资料索引.md
  文案与话术.md
  交接记录.md
```

项目总览必须包含：

- 当前结论
- 项目状态
- 谁负责什么
- 下一步
- 重要路径
- 风险和未验证项

## 文案保存规则

所有可复用文案统一进入 `文案/`，不要散落在聊天记录里。

建议分类：

```text
文案/
  文案总览.md
  官网文案/
  短视频脚本/
  公众号/
  朋友圈/
  社群/
  销售话术/
```

## Agent 内容保存规则

五个 Agent 的长期规则、交接、项目职责统一进入：

```text
智能体/
  智能体总览.md
  Grace.md
  Mia.md
  Atlas.md
  Lisa.md
  Hygeia.md
```

临时执行日志可以留在项目目录；长期规则必须回写到智能体目录。
