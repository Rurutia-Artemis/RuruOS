# 公众号文章入库与排版 Skill｜使用说明

## 它是做什么的？

这是 Mia 把你转发的微信公众号文章整理进 **RuruOS** 的专用流程。

它不会只存一个链接，而是尽量把文章变成适合长期阅读、搜索和复用的 Obsidian 笔记：

- 保留原文来源、作者、发布日期和原链接
- 先做一段简短的「Mia 摘要」与「速读」
- 按原文逻辑恢复标题、段落、列表、代码块和图文节奏
- 下载并本地保存正文重要图片，避免微信图片链接失效
- 对信息图、截图卡片和图中表格，尽量转成可搜索文字、表格或代码块
- 删除广告、二维码、付费课程、关注/点赞/转发引导及页尾推荐
- 入库前查重；完成后检查 YAML、图片、微信热链和页尾噪声

## 你平时怎么用？

最简单：**直接把公众号链接发给 Mia** 就行，例如：

```text
https://mp.weixin.qq.com/s/……
```

不需要再特别写「去广告」「排版」「存进 RuruOS」。Mia 会默认：

```text
查重 → 公开抓取 → 图片本地化 → 富排版 → 去广告 → 写入 RuruOS → 回读验证
```

如果是很多链接，也可以连续发送；Mia 会先排队、去重，再受控并发处理。

## 最终会放到哪里？

完整文章：

```text
RuruOS/20-Reading/Articles/
```

抓取不完整、需要人工判断的内容：

```text
RuruOS/20-Reading/Queue/
```

文章图片会按内容类型保存到：

```text
RuruOS/99-Attachments/Mia/toolwatch/   # AI、工具、Skill、Agent 类
RuruOS/99-Attachments/Mia/design/      # 配色、UI、排版、设计类
```

## 这个文件夹里有什么？

- `SKILL.md`：主规则与完整工作流。
- `references/`：不同文章类型的补充规则，例如信息图、图片卡片、代码截图、批量队列和抓取恢复。
- `scripts/`：公开 HTML 提取、图片卡片提取、队列校对和成品验证脚本。

## 重要边界

- 只处理公开可访问内容：不登录、不用 Cookie、不绕过验证码或访问限制。
- 遇到正文不完整时，会诚实标为待复核，不会假装已完整入库。
- 文中收入、Star 数、效果数据等若没有另行核验，会保留为原文观点并标注。
- 设计、配色、排版类文章可额外产出一份交接素材，留给你的设计角色/流程复用（没有就忽略这条）。

## 这份副本的定位

这是随 RuruOS 公开仓库分发的版本，来自作者的 Hermes 秘书 agent（Mia）在真实使用中沉淀的流程。抓取环节依赖的平台能力（如后台 worker、队列）可用你自己环境里的等价手段替代——核心的排版规则、去噪规则与验证脚本是通用的。
