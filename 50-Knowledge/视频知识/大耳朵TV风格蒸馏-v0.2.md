# 大耳朵TV风格蒸馏 v0.2

> 状态：`preliminary / Grace(OpenAI)`。本版从 v0.1 的小样本观察升级为 **334 条公开视频元数据** 的大样本标题/选题/结构分析。尚未做 Claude 交叉验证，也尚未做批量 ASR，因此“口头禅、脚本文案节奏、镜头语言”仍是待验证项。

## 0. 本版相对 v0.1 的新增结论

- 样本量从 8 条人工样本扩展到 **334 条唯一 BVID**。
- 334 条均通过 `x/web-interface/view` 公共单视频 API 补全成功，`view_api_success=334/334`。
- 频道公开视频总量仍以页面/接口显示的 **990 条** 为上限；本版覆盖约三分之一，足以修正标题与选题模式，但还不是全量。
- 纯终端 WBI/yt-dlp 路径仍不稳定：终端 `arc/search` 会触发 `-352 风控校验失败`，媒体 playurl/yt-dlp 会遇到 HTTP 412。
- 可行主路径变为：**浏览器页面生成签名 WBI URL → 浏览器 fetch 成功 → 本地只读落盘 → 单视频 view API 补元数据**。

## 3. 标题系统 v0.2

### 3.1 统计特征

- 标题长度：中位数约 **49 字**，平均约 **50.9 字**，范围 12-80 字。
- 竖线分隔极高频：`|` 170 条，`｜` 168 条。
- `feat.` 出现 97 条，是频道典型搜索词/补充说明连接器。
- `【】` 出现 111 条，多用于“人群必看 / 买前必看 / 纯VLOG / 年份专题”。
- 感叹号 `！` 出现 168 条，标题情绪强，但正文通常仍需要理性解释。

### 3.2 主标题公式

```text
[强结论/人群标签] + [真实使用时间/数量清单] + [产品/系统/场景] + [明确结果]
｜[搜索关键词串/对比对象/评测维度]｜大耳朵TV
```

### 3.3 最稳定的 6 类标题模板

1. **人群必看型**
   ```text
   【iPhone用户必看】为什么一定要升级到 [系统版本]？| [N]+ 新功能 | 大耳朵TV
   ```

2. **真实使用型**
   ```text
   真实使用[时间]后，发现[产品]最大优点/缺点！feat. 开箱/评测/对比 | 大耳朵TV
   ```

3. **买前必看型**
   ```text
   【买前必看】用了[N年/天]，我为什么放弃[A]，最终选择[B]？｜优缺点总结｜大耳朵TV
   ```

4. **购买指南型**
   ```text
   [N]款不能买！[M]款放心买！苹果[月份/场景]购买指南 | Mac/iPad/iPhone/AirPods | 大耳朵TV
   ```

5. **强情绪 + 清单型**
   ```text
   太强！/严重被低估！/没想到！[产品/系统] [N]大看点！| [关键词1] / [关键词2] / [关键词3]
   ```

6. **人格化 VLOG 型**
   ```text
   【纯VLOG】[生活冲突/挑战/问题] | 韩国 / Q&A / 人生经验 | 大耳朵TV
   ```

### 爆款共性

- 标题直接回答“值不值、该不该买、要不要升级”。
- 有明确时间证据：`1天`、`3天`、`一周`、`2个月`、`69分钟`。
- 关键词堆叠不是噪音，而是 B站搜索/推荐标签：产品名、竞品、评测维度、功能词。
- 情绪词服务于消费判断：`无敌了`、`离不开`、`太值了`、`谦虚了`、`隐藏缺点`。

## 5. 视频结构修正版

### A. 苹果系统/功能更新型

1. **开头 0:00-0:20**：直接给升级结论：一定要升/先别升/严重被低估。
2. **背景 0:20-1:00**：版本、适用设备、发布时间、这次为什么重要。
3. **主体 1:00-7:00**：`N+ 新功能`清单，每个功能都要落到真实使用场景。
4. **风险 7:00-9:00**：稳定性、耗电、兼容、旧机型体验。
5. **结论 9:00-11:00**：谁现在升，谁等等。

### B. 开箱评测/消费决策型

1. **强结论钩子**：这东西到底值不值，先抛结论。
2. **我为什么买/为什么关注**：个人动机带入。
3. **体验维度列表**：屏幕、性能、重量、价格、续航、音质、降噪、发热等。
4. **真实生活场景**：韩国日常、通勤、桌面、旅行、吃饭、家庭环境。
5. **缺点必须讲**：标题越强，正文越需要一个“唯一缺点/最大缺点”。
6. **购买建议收束**：谁该买、谁不该买、何时买。

### C. 纯 VLOG/人格关系型

1. 用冲突问题开场：为什么不戒色、孩子想去泰国、连吃三天、机场 BB。
2. 场景推动而不是参数推动。
3. 中间插入 Q&A/人生经验/家庭对话。
4. 结尾回到轻结论，维持朋友式陪伴感。

## 6. 封面策略 v0.2

v0.1 的封面判断仍成立，但 v0.2 需要强调“标题-封面同构”：

- 标题若是“买前必看/不能买/放心买”，封面应有 **红叉绿勾 + 产品对比 + 大数字**。
- 标题若是“真实使用 N 天”，封面应有 **真人表情 + 产品近景 + 时间数字**。
- 标题若是“系统 N+ 新功能”，封面应有 **iPhone/iPad 界面截图 + N+ 大字 + 版本号**。
- 标题若是 VLOG，封面可降低科技感，强化 **人物/食物/家庭/韩国场景**。
