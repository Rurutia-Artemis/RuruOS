# Dan Koe 外部事实核查

> **结论先行**：本轮能核实的是“哪些公开页面存在、页面上写了什么、YouTube 公开探针在 2026-06-28 返回了什么”。不能核实的是 Dan Koe 的真实收入、真实工时、全平台聚合粉丝、Kortex 经营数字是否经第三方审计。以后引用时必须分清：`公开可见`、`一手自述`、`二手转述`、`分析推断`。

## 1. 证据等级词表

| 等级 | 含义 | 使用方式 |
|---|---|---|
| `publicly-observed` | 公开页面或公开工具在某时点可见 | 可说“页面显示/公开探针返回” |
| `verified-local` | 本地数据可复算、文件可读回 | 可用于本地元数据统计 |
| `self-reported` | 当事人/公司一手声称 | 不等于第三方审计事实 |
| `secondary-reported` | 公众号/媒体转述他人声称 | 只能作营销叙事样本 |
| `interpretation` | Grace/Claude 对材料的分析判断 | 可作为策略假设，不作事实 |
| `pending` | 需要进一步逐句或多源核验 | 不用于最终事实断言 |

Claude 交叉审阅特别提醒：不要用 `thedankoe.com` 证明 `thedankoe.com` 上商业数字为真。只能证明“官网这样写”。

## 3. 对原 Dan Koe 报告的修正

原报告中的核心判断仍成立：

> Dan Koe 模式的核心不是“写作赚钱”，而是长期公开思考积累分发权，再把分发权导向低边际成本产品、高客单服务和可复用系统。

但事实层应按以下方式降级：

1. **收入、工时、全平台粉丝**：仍保持 `secondary-reported / unverified`。
2. **Kortex 相关商业数字**：不是第三方验证，只是 `self-reported`。
3. **YouTube 订阅数**：可写“公开探针返回约 139 万”，但不是审计意义上的总影响力证明。
4. **OpenAI Harness 案例**：文章存在可证；指标是 OpenAI 一手自述。
5. **Mitchell Hashimoto 文章**：原文存在可证；主题概括若要引用，应回到原文逐句核对。

## 4. 可继续采用的策略判断

即使商业数字降级，策略判断不受太大影响：

- 一人公司模仿重点仍是 **分发权 + 产品阶梯 + 交付沉淀**；
- 用户当前更适合从 **B2B 高价值交付物** 开始，而不是从大众课程开始；
- Dan Koe 的可借鉴部分是：用内容测试需求、用受众建立信任、用产品化降低边际成本；
- 不可借鉴的是：用成熟期生活方式叙事误导冷启动期执行。

## 5. Claude 交叉审阅摘要

Claude 的核心意见：

> 整体方法论克制且诚实。需要收紧三处：第一，Dan Koe 部分大量证据是“官网证官网”，必须把“页面存在”与“内容为真”分开；第二，OpenAI 的 0 行手写代码/百万 LOC 也是公司一手自述，不因文章真实而升级为外部事实；第三，公众号关于收入、工时、全平台粉丝的叙事仍是二手营销叙事，不能写成 verified。
