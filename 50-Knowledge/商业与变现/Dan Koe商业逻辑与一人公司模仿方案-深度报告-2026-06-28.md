## 3. Dan Koe 模式的去营销化版本

公众号文章的外层叙事是：

> 普通人 → 深度写作 → 多平台分发 → 产品化 → 每天 4 小时 → 年入 200 万美元。

更严谨的模型应改写为：

```text
长期思考与写作
  → 内容资产
  → 分发权 / 受众信任
  → 可反复触达的流量池
  → 低边际成本产品 + 高客单服务 + 软件/订阅
  → 收入反馈与社会证明
  → 更强信任与更大分发权
  → 飞轮继续转动
```

### 3.1 真正的资产不是“知识”，而是“分发权”

知识产品本身越来越容易复制。AI 可以生成课程大纲、电子书、模板、脚本。真正难复制的是：

- 你是否持续被一群人看见；
- 他们是否相信你对某个问题有判断力；
- 你是否能反复触达他们；
- 他们是否愿意把你的内容转发给类似的人；
- 你是否能把注意力导向付费产品。

所以 Dan Koe 的商业逻辑核心不是“写出知识”，而是：

> **用长期公开思考换取分发权，再用产品阶梯变现分发权。**

### 3.2 “一次创作，无限出售”只说对了一半

数字产品确实有低边际成本：课程、电子书、模板、newsletter、社群内容都可以反复出售。但它们不是零成本：

- 内容会过时，需要更新；
- 市场会拥挤，需要持续差异化；
- 算法会变化，需要重新适配分发；
- 用户会审美疲劳，需要更强的案例和结果；
- 产品销售仍需要持续获客。

更准确的表述是：

> **生产边际成本降低了，但分发和维护成本并没有消失。**

### 3.3 “推特测反响”不是最终验证

文章 A1 说：先发短内容，看哪条数据好，再扩成长文和产品。这个思路有用，但不够硬。

点赞、收藏、转发只能说明“人们愿意看”，不能说明“人们愿意付费”。更强验证顺序应是：

1. 内容数据：是否有人停留、收藏、转发；
2. 邮件/私信：是否有人主动问你怎么做；
3. 候补名单：是否有人留下联系方式；
4. 预售：是否有人提前付款；
5. 交付复购：是否有人获得结果后继续买。

因此，A2 的“先收钱，再动手”比 A1 的“先测内容反响”更接近商业真相。

## 4. 商业飞轮：Dan Koe 到底怎么赚钱

可以拆成五层。

### 4.1 输入层：深度思考与个人问题域

他不是随便追热点，而是围绕一组长期问题反复输出：

- 如何成为一人公司；
- 如何通过写作和个人品牌变现；
- 如何设计生活、工作、学习系统；
- 如何用数字产品摆脱传统工时收入。

这类主题有一个共同点：受众不仅买知识，也买身份想象：

> “我也想成为一个自由、独立、高收入的一人公司。”

这让内容天然带有强转化潜力。

### 4.2 分发层：免费内容建立信任

免费内容的作用不是直接赚钱，而是：

- 让陌生人知道你；
- 让目标人群反复看到你；
- 用持续输出证明你不是一时兴起；
- 用观点筛选同频受众；
- 把受众导向邮件列表、产品页、社群或软件。

### 4.3 产品层：价格阶梯

Dan Koe 式个人商业通常不是只有一个产品，而是价格阶梯：

| 层级 | 形式 | 作用 |
|---|---|---|
| 免费层 | 推文、长文、视频、newsletter | 获客、建立信任、测试选题 |
| 低价层 | 电子书、模板、小课 | 首次付费，验证购买意愿 |
| 中价层 | 系统课程、会员社群 | 提高客单价和复购 |
| 高价层 | 咨询、训练营、企业服务 | 最大化信任转化 |
| 杠杆层 | 软件、订阅、授权 | 从人力交付转向产品化资产 |

### 4.4 反馈层：市场反应反过来塑造内容

真正的飞轮不是“写完就卖”，而是：

```text
内容发布 → 数据反馈 → 用户问题 → 产品假设 → 付费验证 → 交付案例 → 新内容 → 更高信任
```

这里最关键的是：每一轮交付都要沉淀成可复用资产：

- SOP；
- 模板；
- 案例；
- 清单；
- 提示词；
- 工具链；
- 方法论图谱。

### 4.5 复利层：受众资产 + 知识资产 + 系统资产

真正可复利的是三类资产：

1. **受众资产**：越来越多目标人群知道你、信任你；
2. **知识资产**：你的文章、框架、案例、课程持续被引用；
3. **系统资产**：你的 SOP、工具、自动化、AI Agent 帮你降低交付成本。

只做内容而不做产品，难赚钱。只做产品而不做分发，难获客。只做服务而不沉淀系统，难复利。

## 5. 这套模式的硬伤与风险

### 5.1 幸存者偏差

我们看到的是 Dan Koe 成功后的叙事，看不到大量同样写作、同样做课程、同样公开构建但没有成功的人。

因此不能推导出：

> 只要每天写作，就能年入百万美元。

只能推导出：

> 对已经形成受众资产的人，写作 + 产品阶梯 + 分发飞轮是一种高杠杆变现方式。

### 5.2 冷启动成本被淡化

“每天 4 小时”很可能是飞轮成熟后的状态，不是冷启动状态。冷启动期可能需要：

- 长期高频输出；
- 大量失败内容；
- 社群互动；
- 产品迭代；
- 销售页打磨；
- 客户支持；
- 退款和差评处理；
- 持续学习和研究。

如果照着成熟期生活方式模仿，容易失败。

### 5.3 分发平台风险

Dan Koe 的模式依赖 X/Twitter、YouTube、newsletter 等平台。平台变化会影响：

- 推荐算法；
- 内容触达；
- 账号安全；
- 广告和销售规则；
- 受众迁移。

所以一人公司不能只押单一平台，虽然冷启动期应该先专注一个主阵地。

### 5.4 知识产品同质化

AI 让低质量信息产品泛滥。未来信息产品要成立，必须具备至少一种优势：

- 真实案例；
- 独特经验；
- 可验证结果；
- 高质量社群；
- 工具/SOP 配套；
- 明确的转化结果。

否则只是“AI 生成课程”的又一个副本。

## 6. 你应该模仿什么，不该模仿什么

### 应该模仿

1. **用写作作为底层表达与思考系统**；
2. **先公开输出，再用市场反馈修正方向**；
3. **把交付过程沉淀成 SOP、模板、课程、工具**；
4. **围绕一个窄问题域长期建立信任**；
5. **用 AI 做生产加速器，而不是替代判断**；
6. **设计价格阶梯，而不是只有一个产品**。

### 不该模仿

1. 不要直接模仿“做个人成长内容”；这是红海；
2. 不要一开始就做大众课程；你还没有大受众；
3. 不要把“每天 4 小时”当启动期标准；
4. 不要把点赞当需求验证；
5. 不要同时做五个平台；
6. 不要同时做产品、服务、涨粉、软件、社群，导致每条线都不够深。

## 7. 最适合你的模仿方向：B2B 高价值交付物先行

基于你现有的信息流、Hermes 多 Agent、Mia/Obsidian 管道、Claude Code/Agent/Harness 兴趣，更适合的切入不是“做个人成长博主”，而是：

> **AI 工作流 / Agentic Harness / Obsidian 知识系统落地服务。**

### 7.1 推荐定位

候选定位：

1. **给独立开发者和小团队搭 Claude Code / Codex / Hermes 工作流**  
   交付：AGENTS.md、CLAUDE.md、Skills、Hooks、测试/验证流程、项目知识图谱。

2. **给内容创作者/研究型个人搭 Obsidian + AI 信息流系统**  
   交付：微信/网页/视频 intake、自动摘要、分类、主题地图、周报/月报。

3. **给设计/内容团队搭 AI 素材库与灵感管道**  
   交付：素材采集、标签体系、Mia 队列、Obsidian 知识库、可复用 Prompt/Skill。

最建议从第 1 或第 2 个开始，因为它们更容易卖 B2B/高客单交付物。

### 7.2 为什么不要先做 C 端课程

C 端课程需要：

- 大量受众；
- 低客单高销量；
- 持续客服；
- 强营销；
- 很多用户成功案例。

你现在更适合：

> 少量高价值客户 × 明确交付物 × 可沉淀 SOP。

例如：

- 3 个 pilot 客户 × ¥8,000 = ¥24,000；
- 或 3 个 pilot 客户 × $1,000 = $3,000；
- 之后把交付 SOP 产品化。

这比一开始卖 ¥99 课程现实得多。

## 8. 90 天行动方案

### 总原则

- **现金引擎**：先卖服务/交付物，拿真实客户和案例；
- **资产引擎**：把服务过程写成内容、SOP、模板、课程；
- **只选一个主平台**：不要五个平台同时发；
- **先窄后宽**：细分到让目标客户觉得“这就是给我做的”。

### 第 1–2 周：定位与预售

目标：确认一个窄 offer。

动作：

- 写一句话定位：  
  > 我帮【某类人】在【7–14 天】内搭好【某个 AI 工作流/知识系统】，让他们【节省时间/提升交付/减少 Agent 错误】。

- 列 20 个潜在线索：熟人、熟人的熟人、小团队、创作者、独立开发者。
- 设计一个 pilot 交付包：
  - 交付周期：7–14 天；
  - 价格：¥3,000–¥12,000 或 $500–$2,000；
  - 交付物：配置文件、SOP、Obsidian 模板、自动化脚本、使用说明、一次复盘。
- 直接发 20 条私信/邮件，目标不是点赞，是拿到 3 个深聊和 1 个预付款。

### 第 3–4 周：交付第一个客户

目标：完成第一个真实案例。

动作：

- 不追求完美，追求可用；
- 每天记录：问题、决策、配置、踩坑；
- 用 Claude 整理成 SOP；
- 用 ChatGPT 快速生成对外说明文案；
- 交付后要客户反馈与可匿名案例。

沉淀资产：

- 一份交付 checklist；
- 一份前后对比案例；
- 一篇公开复盘文章。

### 第 5–8 周：重复交付 2–3 次

目标：把即兴服务变成可复制服务。

动作：

- 第二个客户开始按 checklist 做；
- 第三个客户开始涨价或缩短交付周期；
- 每次交付后更新 SOP；
- 把重复部分整理为模板包。

内容策略：

- 每周 3–4 条内容；
- 只发一个平台；
- 只展示过程、具体问题、具体前后对比；
- 不晒空泛收入。

### 第 9–12 周：产品化与价格阶梯

目标：从服务里拆出低价产品。

可能产品：

- Obsidian AI 信息流模板包；
- Claude Code/Hermes Agent 工作流 checklist；
- Harness Engineering 实战小课；
- 交付型咨询诊断包；
- 付费社群/工作坊。

价格阶梯示例：

| 层级 | 产品 | 价格 |
|---|---|---:|
| 免费 | 公开文章/案例拆解 | ¥0 |
| 低价 | 模板包/checklist | ¥99–499 |
| 中价 | 2 小时诊断/工作坊 | ¥1,000–3,000 |
| 高价 | 7–14 天落地服务 | ¥8,000–30,000 |
| 长期 | 月度维护/顾问 | ¥3,000–15,000/月 |

### 第 13 周：复盘

复盘四个指标：

1. 收入：实际收款多少；
2. 线索：多少人深聊；
3. 转化：线索到付款比例；
4. 资产：沉淀了多少 SOP、模板、案例、文章。

根据数据决定：

- 加倍当前定位；
- 缩窄目标客户；
- 调整交付物；
- 或终止该方向。

## 9. Claude 与 ChatGPT 的分工建议

### Claude 更适合

- 长文报告；
- SOP 整理；
- 复杂技术文档；
- Claude Code / Agent / Harness 工作流审阅；
- 从客户交付中抽象流程；
- 对方案做严谨反驳。

### ChatGPT 更适合

- 快速发散想法；
- 营销标题和短文案变体；
- 结构草稿；
- 竞品/市场角度快速拆解；
- 把长内容改成多平台短内容。

### 关键原则

不要因为“同时用两个 AI”导致工作流分裂。建议：

> Claude 做深度和严谨性，ChatGPT 做速度和变体；最终判断由你负责。

## 11. 最终建议

如果你真的想模仿 Dan Koe，不要从“我要做一个个人品牌”开始，而要从这个问题开始：

> **我能为哪个具体人群，在 7–14 天内交付一个清晰、可验证、愿意付费的结果？**

你的第一阶段不是成为 Dan Koe，而是拿到：

- 3 个真实客户；
- 3 个可匿名案例；
- 1 套可复用 SOP；
- 10 篇过程型内容；
- 1 个可预售的小产品。

这才是适合你当前阶段的 Dan Koe 模式。
