# 接入 AI 指南

RuruOS 的数据层是纯 markdown + frontmatter，所以**任何**能读写文件或输出文本的智能体都能接入。区别只在接法。

## 先分清：哪些功能联网免费、哪些消耗 AI 额度

| 功能 | 走什么 | 花钱吗 |
|---|---|---|
| 搜剧、选海报、分季、集数 | TVmaze / 豆瓣 免费公开 API | 不花 |
| 天气卡 | Open-Meteo 免费 API | 不花 |
| 农历、小黄历、时钟 | 本地推算，零网络 | 不花 |
| 「补剧集资料」（建档后自动补中文名/评分/简介） | 本机 claude CLI | 消耗 Claude 额度 |
| 「搜评分」「收录简介」按钮 | 本机 claude CLI | 消耗 Claude 额度 |
| 「AI 浓缩」「AI 清杂」按钮 | 本机 claude CLI | 消耗 Claude 额度 |
| 中文剧名转译兜底（豆瓣也搜不到时才触发） | 本机 claude CLI（haiku，很便宜） | 消耗 Claude 额度 |

**AI 浓缩**：扫描已读且未浓缩的文章，每篇提炼 5-10 条要点写成笔记放进 `50-Knowledge/` 对应主题目录，并给原文标记 `distilled: true`。原文不动。
**AI 清杂**：找出文章库里的垃圾（乱码、残缺、纯广告），移进库内 `.trash/` 文件夹——**不会永久删除**，拿不准的只做标记。两个按钮都在后台跑，跑完弹通知。

## 接法一：Claude Code（推荐）

1. 安装 [Claude Code](https://claude.com/claude-code) 并登录
2. 完事了。两个通道自动打通：
   - **插件按钮**：中枢里所有 AI 按钮自动激活（插件按常见路径探测 `claude` 命令；装在特殊位置的话，在 `main.js` 顶部「公开版配置区」的路径数组里加一行）
   - **对话整理**：终端 `cd` 到库目录跑 `claude`，它会自动读 `CLAUDE.md` 和 `AGENTS.md`，直接说「把这篇文章收进库里」「帮我把这部剧建档」即可

## 接法二：Codex CLI

Codex 原生读取 `AGENTS.md`（本库已备好）。终端 `cd` 到库目录跑 `codex`，直接下指令，写入规则自动生效。插件内的 AI 按钮不认 Codex——按钮功能想用还是得装 Claude Code，或者无视按钮、全靠对话完成同样的事。

## 接法三：其他本地 CLI 智能体（Gemini CLI 等）

在它的规则文件（如 `GEMINI.md`）里写一行「写入本库前先完整阅读 AGENTS.md 并严格遵守」，或者每次会话开头让它先读 `AGENTS.md`。

## 接法四：聊天型 / 云端智能体（Hermes、Grok、ChatGPT、Coze/扣子…）

它们碰不到你的文件系统也没关系：

1. 把根目录的 **`给智能体的说明书.md`** 全文设为它的系统提示词（或上传为知识文件）
2. 之后把文章链接、剧名、账目丢给它，让它**输出符合规范的完整 markdown 文件内容**（含 frontmatter）
3. 你把输出保存为 `.md` 文件放进说明书指定的目录——文件一落地，中枢界面自动更新

如果你的智能体平台带文件系统/仓库写入能力（比如能操作 git 仓库的云端 agent），直接让它按说明书写入对应目录即可，效果与本地 CLI 相同。

## 想换成别家 AI 驱动插件按钮？

插件的 AI 调用集中在 `main.js` 的三处：`runClaudeTask()`（浓缩/清杂）、`runClaude()`（补资料/搜评分/收录简介）、`askClaude()`（快问快答）。它们本质都是「spawn 一个命令行进程 + 传 prompt + 收 stdout」，替换成任何提供 CLI 的模型服务都不难——欢迎 PR，但注意改动要过 `DESIGN.md` 的验收习惯：改完真机截图自验。
