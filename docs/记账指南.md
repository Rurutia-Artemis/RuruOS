# 记账指南（写给 Agent）

> **这份文档写给替用户记账的 AI。** RuruOS 刻意不做记账表单——**Agent 就是记账入口**：用户对你说一句「昨天便利店花了 128」，你写一个 markdown 文件，中枢自动把它渲染成杂志级的账本页面。
> schema 的最高权威是根目录 `AGENTS.md`（本文与之完全一致，只是展开讲）；喂给聊天机器人的精简版在《给智能体的说明书.md》⑤⑥ 节。

## 数据流一句话

```
用户一句话 → 你写 md 文件 → 中枢自动渲染
```

只有两种对象，都是纯 markdown：

| 对象 | 是什么 | 放哪 |
|---|---|---|
| **流水**（`type: transaction`） | 日常收支，一笔一个文件 | `40-Life/Finance/流水/` |
| **账本**（`type: ledger`） | 一个项目一本的持续账（创业、旅行基金、分期…） | `40-Life/Finance/` 根目录 |

## 一、记一笔流水（最高频操作）

文件名固定 `YYYY-MM-DD 标题.md`，如 `2026-07-12 咖啡店招牌制作.md`：

```yaml
---
type: transaction
title: "一句话说明这笔账"
date: 2026-07-12        # 发生日期
amount: 190000          # 数字，一律正数（方向靠 direction 表达）
currency: JPY           # JPY | CNY | AUD | USD
direction: 支出          # 支出 | 收入
category: 项目           # 餐饮 | 交通 | 购物 | 订阅 | 居住 | 娱乐 | 医疗 | 投资 | 项目 | 其他
ledger: ""              # 关联项目账本的标题（如 海边咖啡店创业），日常消费留空
note: ""
added: 2026-07-12
---
```

正文可以留空。写完它会立刻出现在中枢**账目卡**：本月按主币种汇总、分类彩签、点击可进流水明细。

## 二、开一本账本 / 更新账本

```yaml
---
type: ledger
title: "海边咖啡店创业"
currency: JPY
total: 2865000          # 当前累计金额。余额/欠款类直接写负数（如 -128000），中枢自动红色显示＋标注「当前为欠款」
status: active          # active | closed
created: 2026-07-01
updated: 2026-07-12
---
```

**铁律：正文的明细变了，`total` 和 `updated` 必须同步变。** 中枢账目卡和账本详情页的 hero 大金额直接读 `total`，不会自己算——正文与 frontmatter 对不上就是错账。

单笔支出同时属于某个项目时，两边都要动：流水照记（`ledger` 字段填账本标题）＋ 账本明细表加一行 ＋ `total` 更新 ＋ 更新记录加一条。

## 三、呈现契约：写什么形状，渲染什么效果

账本详情页不是普通 markdown 预览。中枢按 `## ` 二级标题分段，逐段识别你写的内容并变形成对应的视觉组件。**想保持这套界面的美观，照下表的形状写就够了**——你负责数据形状，中枢负责好看：

| 你在正文里写的 | 中枢渲染成 |
|---|---|
| `## 更新记录` ＋ 无序列表 | 竖向时间线（糖果圆点＋日期章） |
| 表格里有一列表头叫「占比」 | 分段占比条（签名元素：六色轮转、加载时动画展开、图例带金额） |
| 两列表格且不超过 6 行 | 一排统计胶囊 |
| 其他任何表格 | 对账单式明细表（斑马纹、吸顶表头、金额琥珀色、合计行加重） |
| 普通文字 / 引用 | 杂志排版正文 |

各组件的形状细则：

- **占比条**：占比列写百分数（`63%`）；表格里最好同时有一列表头含「金额 / 累计 / 小计」（或币种码），图例才能显示每段的金额；写「合计」的行自动跳过，不用删。
- **明细表**：金额写千分位逗号（`1,800,000`）；**金额列的表头要带「金额」两个字或币种码**，这一列才会用琥珀色数字体；最后加总行第一格写「合计」，整行自动加重；金额列出现负数自动显示为退款绿——退款/返现就直接写负数。
- **时间线**：一行一条 `- 2026-07-12：桌椅入场，累计支出 ¥2,865,000`（日期后带中英文冒号都行，带冒号日期会独立成章；不带也能渲染）。**新条目加在列表最上方**（文件里新→旧），页面时间线自动按老→新自上而下展示。
- 表格一律用纯 markdown 管道表。**不要写 HTML、不要嵌套表格、不要合并单元格**——解析器只认标准表。

## 四、新账本推荐骨架（直接抄）

```markdown
## 出资构成

| 出资人 | 金额 | 占比 |
|---|---|---|
| A | 1,800,000 | 63% |
| B | 1,065,000 | 37% |

## 支出明细

| 日期 | 项目 | 金额 | 备注 |
|---|---|---|---|
| 2026-07-01 | 店面押金 | 900,000 | 两个月押金 |
| 合计 | | 900,000 | |

## 更新记录

- 2026-07-01：签约，账本建立
```

三段可按需增删：没有多人出资就删「出资构成」；分期类账本可加一个两列小表（`| 已还期数 | 12/24 |` 这种）自动变统计胶囊。库里的三本「示例账本｜…」就是活的样板，拿不准就打开对照。

## 五、常见任务菜谱

| 用户说 | 你做 |
|---|---|
| 「昨天便利店花了 128」 | 写一个流水文件（category: 餐饮，date 填昨天） |
| 「咖啡店项目又付了 19 万招牌钱」 | 流水（ledger 填账本名）＋ 账本明细表加行 ＋ total 加 190000 ＋ 更新记录加条 |
| 「工资到账 32 万」 | 流水，direction: 收入 |
| 「上次那笔退了」 | 日常消费：新记一笔 direction: 收入 的冲销流水；账本内：明细表加负数行（自动退款绿）＋ total 相应减 |
| 「帮我开一本旅行基金的账」 | 按第二、四节建账本，total 从 0 或首笔起记 |
| 用户发来表格/截图 | 数据转成上面的格式入库；原始件的占位说明放 `40-Life/Finance/原始表格/` |
| 「这个月花了多少？」 | 读流水直接回答——不用建任何新文件 |

## 六、红线与交付自查

- [ ] `total` 与正文合计对得上（这是账，不是笔记）
- [ ] 金额一律数字，不带货币符号进 frontmatter；币种只用 `JPY | CNY | AUD | USD`
- [ ] 删除/冲销不硬删文件：进 `.trash/` 或写冲销流水
- [ ] 用户手写的 `note` 字段永远保留勿动
- [ ] 表格是纯管道表，没有 HTML
- [ ] 干完活提醒用户：中枢账目卡 → 点进账本，看一眼渲染效果
