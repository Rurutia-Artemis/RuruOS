## 一句话结论

**自有日本实体卡店是整个方案的地基：卡图以「在售商品图」身份免费展示，挂真实价格并深链到公司官网的单卡商品页；付费（IAP）只解锁我们自己的原创内容（测试/测评文字），永远不锁卡图。**

## 1. 为什么日本 Oripa 平台能用官方卡图

- **法律依据：日本著作权法第 47 条の 2**（2009 年新增）。实物（美术/摄影作品的原件或复制品）的**所有者或有转让权原的人**（及其**受托者**），为「转让的要约」目的，可以未经许可复制并上网展示该作品图像。立法背景就是网拍/网购。
  - 条文解说：[虎ノ門法律特許事務所](https://chosakukenhou.jp/bijutsu-fukusei/)、[条文全文解说 note](https://note.com/copyrights/n/na2a337972e80)
- **权利消尽**：正版卡首次售出后，实体卡转售不需要权利人许可（二手卡店、Mercari 的合法性来源）。
- **像素限制**（施行令）：无防复制措施 ≤ 32,400 像素（约 180×180）；有措施 ≤ 90,000（约 300×300）。
- **明文排除数字销售**：「デジタル画像や電子書籍のネットでのダウンロード販売などには本条は適用されません」——付费数字内容里放卡图 = 豁免不适用。

### Oripa 们自己在裸奔的部分（不能当依据抄）

- 高清大图 + 抽卡动画演出，远超像素上限；
- 部分平台无现货先卖后进（律师警告会引爆富くじ风险，[LEON 律所分析](https://legal-leon.jp/column/1102/)）；
- 宝可梦官方多次[注意喚起](https://www.pokemon-card.com/info/003978.html)，态度是「法律上难打二手实物 + 容忍」，绝非许可；
- 賭博罪/景表法始终是灰色；需古物商許可。
- **App Store 上架 ≠ 合法**：苹果不主动查 IP 授权，权利人一封投诉就下架，不看 app 大小。

## 2. 为什么纯内容型 App 不行（前车之鉴）

- **Pokellector 案（2014，美国）**：纯卡牌图鉴/收藏 app，被宝可梦以版权+商标起诉。
- **Beckett 杂志案（2010）**：刊登数百张卡图被诉，和解。
- 宝可梦是全行业对卡图管最狠的 IP 方；对数字侵权真的动手（中国《口袋妖怪：复刻》案赔 1.07 亿元）。
- **TPCi 官方明确拒绝受理一切 IP 使用申请**（[官方 FAQ](https://support.pokemon.com/hc/en-us/articles/360000634094)）——"给官方发邮件申请授权"这条路不存在。

## 3. 授权路线盘点（调研结论）

| 路线 | 状态 | 备注 |
|---|---|---|
| 官方授权 | ❌ 死路 | TPCi 不受理；媒体素材包仅限非商业编辑用途 |
| TCGplayer API | ❌ 已关闭 | 2024 起停发新 key（eBay 旗下） |
| eBay Partner Network + Browse API | ✅ 可行 | 条款授权展示 listing 内容用于导流，美版卡主路径 |
| 駿河屋アフィリエイト | ✅ 可行 | トレカ在覆盖品类，二手佣金 4.2%，[申请入口](https://affiliate.suruga-ya.jp/) |
| 乐天/Yahoo!ショッピング API | ✅ 可行 | 联盟 API 直接吐商品图并授权展示 |
| pokemontcg.io / TCGdex 等社区 API | ⚠️ 灰色 | 行业常态但它们自己就没授权，拿不出文件 |
| 与卡店签受托贩卖合同 | ✅ 最干净 | 成为 47条の2 的「受託者」 |
| UGC + 避风港（用户扫自己的卡） | ✅ 结构成立 | DMCA / プロバイダ責任制限法，冷启动慢 |
| 纯数据（卡名/编号/价格） | ✅ 零风险 | 事实不受版权保护，永远的兜底 |

## 4. 我们的定案架构（自有日本卡店版）

```text
每张卡的页面：
 ├── 卡图（免费）＝在售商品图（自己扫描/实拍，不扒官方文件和别人照片）
 ├── 真实价格 + 「购买」→ 深链到公司官网该卡的商品页
 └── 原创内容（人格测试/测评/分析）→ IAP 付费解锁
```

- 实体商品**必须**外部支付（审核指南 3.1.5(a)，禁 IAP）；数字内容**必须** IAP（3.1.1）。两条规则正好各归各位。
- 没货的卡：纯数据位 + 到货提醒/求购登记；进货后图才出现。寄卖（委託販売）是低成本扩卡池的正路。
- 买取（收卡）页面挂卡图不受豁免保护，用纯数据表。
- App 先只上日本区；海外订单由官网承接（店支持海外付款+邮寄）。

### 五条承重墙

1. **有货才有图**——豁免附着在「真实在售的那张卡」上，不附着在卡店身份上。
2. **深链到具体商品页**，售罄同步下架图/换数据位。审核员和律师都会点那个按钮。
3. **价签是真的**——每个价格都是愿意成交并发货的价格。摆高价当内容幌子 = 掏空豁免（"野心定价"可以，"劝退定价"不行；可用「市场参考价 vs 本店价」包装溢价）。
4. **付费墙后面只有自己的文字和数据**——任何"付费看卡图/高清图/完整图鉴"变体都是直接卖宝可梦美术作品，全线红区。
5. **品牌词零出现**：app 名、副标题、关键词、icon、商店截图、IAP 商品名都不带 Pokémon/ポケモン/宝可梦及形象元素（内容正文里指示性使用没问题）。

## 5. Apple 审核作战手册

- **个人开发者账号 + 公司授权书**：合法且有过审先例（开发者论坛案例：公司出授权声明后 5.2.1 通过）。授权书内容：公司委托个人开发运营、销售公司持有的正版实体卡、图像均为在售商品图。日英双语。
- **卖家主体必须是公司**：特商法表記、发货、收款都走公司；个人只是运营受托方。中期给公司注册组织开发者账号（需 D-U-N-S），app 可转移。
- **Metadata 是最大被拒点**（Ravensburger "Memory" 案例：一个词就发起下架；metadata rejected 不用重传 binary）。分类选 Shopping。
- **提前放进 App Review Information**：授权书、营业执照、古物商許可、实体店照片、现有网店链接。
- **审核 demo**：每张展示的卡都有价格和可购买状态，购买流程走得通，付费墙后能看到原创内容（给解锁演示账号）。避免大片"有图无货"条目。
- **被 5.2.1 问询**：Resolution Center 回复口径——"我们是销售正版实体卡的零售商，图像为在售实物的商品展示，与其他平台销售正版商品性质相同"，附文件。**铁律：不伪造文件（3.2(f) 封号）、被拒不换新账号重交（关联连坐）**。
- 避开 4.2（最低功能性）：浏览/搜索做原生，只把结账交给网店，不做纯网页壳。
- Oripa 抽卡玩法若以后要加，是另一套审核逻辑（概率公示、未成年人保护），先以纯零售+内容形态站稳。
