## 使用原则

1. 市场调研默认优先 Perplexity，尤其是需要来源和竞品价格时。
2. Perplexity 输出标记为 `Perplexity-cited`，不自动等同 verified。
3. 关键商业判断需要 Grace 综合，必要时 Claude-reviewed。
4. 不把 Perplexity API key 写入聊天、Obsidian、prompt 或报告。
5. API 成本可控：普通问题用 `sonar-pro`，重要深研用 `sonar-deep-research`。
