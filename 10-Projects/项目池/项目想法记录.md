### 设计类 Agent / 工具：自动调研并提炼设计方向

- 初始描述：想做一个偏设计类的 Agent / 工具。
- 关键前提：它必须先能理解审美、懂审美，而不是只机械搜索和截图。只有先建立对“美感、风格、视觉质量、设计判断”的理解，后面根据截图找规律、总结方向、继续执行设计才有意义。
- 核心流程：
  1. 先理解需求背后的审美目标：气质、风格、调性、视觉偏好、目标用户感受。
  2. 根据我的设计需求，自动去搜索各大设计类网站、案例库、竞品或灵感来源。
  3. 截取相关页面、案例、组件、视觉风格等截图。
  4. 基于审美判断，从收集到的截图和案例中总结规律，提炼出我们想要的设计方向。
  5. 再根据总结出的方向继续执行设计操作，例如生成方案、做页面、出视觉稿或给设计规范。
- 可能价值：把「理解审美 → 找参考 → 截图归档 → 分析风格 → 输出设计方向 → 执行设计」这条链路自动化。
- 待明确：
  - 如何让 Agent 学会/对齐我的审美？需要我的偏好样本、喜欢/不喜欢案例、品牌关键词、风格板吗？
  - 主要面向网页设计、App 设计、品牌视觉，还是产品 UI？
  - 需要接入哪些设计来源：Dribbble、Behance、Pinterest、Mobbin、Awwwards、竞品官网等？
  - 最终产出要是什么：Moodboard、设计报告、Figma 草稿、HTML 原型，还是可直接交给设计师/开发的规范？
  - 是否需要自动截图和分类存档？

### 日本本地商家AI收费项目：无官网小店 → 轻官网/MEO运营

- 初始描述：在日本用 Google 地图搜索餐厅等本地店铺，找出没有官网或官网很旧的店，先用 AI 做一个官网样稿和效果展示，给店主看；如果对方觉得有价值，再转成付费服务。
- 初步判断：可行，但不要只卖一次性官网，更适合包装成「轻官网 + Google Business Profile / MEO + 口コミ/投稿运营」的持续服务。
- 推荐切入行业：餐厅、美容院/美甲/沙龙、小型旅馆/民宿、观光体验店。
- 核心卖点：不是“AI做网站”，而是帮店铺提升 Google 地图曝光、预约/来客转化、多语言介绍和线上可信度。
