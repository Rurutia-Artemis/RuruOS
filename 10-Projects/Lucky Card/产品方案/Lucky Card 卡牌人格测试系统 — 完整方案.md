## 一、心理学理论基础与维度体系设计

### 1.1 参考的心理学模型

我们的维度体系综合了三个经典理论的优点，取其适合"娱乐性人格测试 + 卡牌匹配"的部分：

#### 大五人格模型（Big Five / OCEAN）

心理学界公认最稳定的人格分类框架，包含五个维度：

- **开放性** (Openness): 想象力、好奇心、审美偏好
- **尽责性** (Conscientiousness): 自律、秩序感、目标导向
- **外向性** (Extraversion): 社交能量、主张性、冒险精神
- **宜人性** (Agreeableness): 同理心、信任倾向、合作意愿
- **神经质** (Neuroticism): 情绪稳定性、压力反应

每个维度下还有 6 个子维度（facets），用于精细测量。我们的系统借鉴其"连续光谱"思想——不做非此即彼的分类，而是在每个维度上给出一个连续分数。

#### 荣格原型理论（12 Archetypes）

Carl Jung 提出的 12 种心理原型（英雄、魔法师、探索者、叛逆者、照顾者、创造者、统治者、智者、天真者、爱人者、弄臣、普通人），被广泛用于品牌营销和娱乐测试。

其优点是：每种原型都有"动机 + 恐惧 + 天赋"的完整叙事结构，非常适合用来写人格解读文案。我们的 16 种人格类型命名和描述风格借鉴这套叙事框架。

#### MBTI 的产品设计方法论

MBTI 在科学界争议较大，但作为产品它非常成功。我们借鉴的不是它的理论，而是它的**产品设计逻辑**：

- 二元选择降低认知负担（每道题只需在两端中选择）
- 结果以"类型标签"呈现（人天然喜欢给自己贴标签）
- 提供正面描述为主、负面描述为辅的解读（让用户觉得"说的就是我"）
- 类型之间有关系网络（互补型、对立型），促进社交传播

### 1.2 Lucky Card 维度体系（6 维度）

综合以上理论，并结合卡牌数据能提供的结构化字段，我们定义以下 6 个核心维度：

|编号|维度名称|英文代号|低分端（0）|高分端（10）|对应大五|对应卡牌属性|
|---|---|---|---|---|---|---|
|D 1|力量倾向| `power` |温和、回避冲突|强势、直面对抗|外向性的主张性子维度|攻击力/伤害值|
|D 2|神秘感| `mystery` |坦率、透明|深沉、难以捉摸|开放性的幻想子维度|暗属性/超能力属性|
|D 3|守护本能| `protection` |优先自保|优先保护他人|宜人性的利他子维度|HP/防御力/防御效果|
|D 4|思维方式| `mind` |直觉驱动、感性|策略优先、理性|开放性的智识子维度|效果怪兽/特殊能力|
|D 5|秩序感| `order` |自由、反叛、混沌|规则、纪律、秩序|尽责性|光属性/正规军种族|
|D 6|社交能量| `social` |独行、内向|群体、外向|外向性的群居性子维度|群体攻击/辅助效果|

**为什么是 6 个维度？**

- 6 维向量在 0~10 的范围内，理论组合空间为 11^6 ≈ 177 万种，足够区分上万张卡牌
- 与大五模型的 5 维相比，我们将外向性拆成了"力量倾向"和"社交能量"两个独立维度，因为在卡牌世界里"强势攻击"和"群体协作"是两个完全不同的概念
- 每个维度都能直接从卡牌的结构化数据中提取信号，不存在"悬空维度"

#### 简单模式向量计算

```
用户选择的所有关键词的权重向量 → 逐维度加总 → 除以选择总数 → 归一化到 0~10
```

例如用户在 8 屏中分别选了 [思考者, 夜空, 很聪明, 解决难题, 不被理解, 古老遗迹, 用脑子赢, 魔法]，计算过程：

1. 把 8 个权重向量逐维度相加
2. 除以 8（选择总数）得到平均值
3. 对每个维度做 min-max 归一化到 0~10

最终输出一个 6 维用户向量，如 `{power: 3.8, mystery: 7.2, protection: 3.5, mind: 8.5, order: 4.0, social: 3.1}`

## 三、卡牌数据结构分析与映射逻辑

### 3.1 宝可梦 TCG API 数据结构

|字段|示例|用途|
|---|---|---|
| `name` |"Venusaur-EX"|卡名，用于 AI 打标签和展示|
| `supertype` |"Pokémon" / "Trainer" / "Energy"|超级类型，筛选只保留 Pokémon 卡|
| `subtypes` |["Basic", "EX"]|子类型，区分进化阶段和特殊类型|
| `hp` |"180"|生命值，映射到 protection 维度|
| `types` |["Grass"]|属性类型（11 种），核心映射字段|
| `attacks` |[{name, cost, damage, text}]|攻击招式名称/描述/伤害，映射多维度|
| `weaknesses` |[{type: "Fire", value: "×2"}]|弱点|
| `resistances` |[{type: "Fighting", value: "-30"}]|抵抗力|
| `retreatCost` |["Colorless", "Colorless"]|撤退费用，间接反映体型/笨重度|
| `abilities` |[{name, text, type}]|特殊能力描述|
| `rarity` |"Rare Holo"|稀有度|
| `flavorText` |"A brutal Pokémon..."|风味文字，AI 打标签的重要输入|
| `images.large` |URL|高清卡面图|

|---|---|---|---|---|---|---|
|Fire|+3|0|0|0|0|+1|
|Water|0|+1|+2|+1|+1|+1|
|Grass|0|+1|+2|+1|+2|+2|
|Lightning|+2|0|0|+1|0|+1|
|Psychic|0|+3|+1|+3|0|0|
|Fighting|+3|0|+1|0|+2|+1|
|Darkness|+2|+3|0|+1|-1|0|
|Metal|+1|0|+3|+1|+3|0|
|Dragon|+3|+2|+1|+1|+1|0|
|Fairy|0|+2|+2|+1|+1|+2|
|Colorless|+1|0|+1|0|+1|+2|

**HP 映射**：

- HP < 60 → protection += 0
- HP 60~90 → protection += 1
- HP 100~140 → protection += 2
- HP 150~200 → protection += 3
- HP > 200 → protection += 4

**攻击伤害映射**（取最高伤害值）：

- damage < 30 → power += 0
- damage 30~60 → power += 1
- damage 70~100 → power += 2
- damage 110~150 → power += 3
- damage > 150 → power += 4

**进化阶段映射**：

- Basic → social += 1（独立性）
- Stage 1 → mind += 1（成长型）
- Stage 2 → mind += 2, order += 1（成熟型）

### 3.3 AI 打标签策略（规则层 + AI 层）

#### 第一层：规则层

将上述映射表写成一个脚本，对每张卡的结构化数据进行自动打分。规则层处理完后，每张卡就有了一个粗略的 6 维向量。

具体流程：

1. 初始化 6 维向量为 [0, 0, 0, 0, 0, 0]
2. 根据属性/类型/种族等结构化字段累加分数
3. 将最终分数归一化到 0~10

这一层免费、快速，可以在几分钟内处理完所有卡牌。

#### 第二层：AI 精修层

规则层无法处理的信息：

- 卡名的意象（如"黑暗魔术师"暗示 mystery）
- 效果描述的语义（如"保护你的怪兽"暗示 protection）
- 风味文字的氛围（如"传说中出现在暴风雨中的鸟"暗示 power + mystery）

用 Claude API 对规则层结果进行微调。**批量处理策略**：每次 API 调用处理 15~20 张卡，节省成本。

## 四、16 种人格类型定义

### 4.1 类型体系设计原则

16 种类型来自 6 维空间的聚类划分。具体方法是将 6 个维度两两组合，形成 4 组"主导维度对"，每组再分高/低，共 16 种：

- **核心轴 1**: power vs protection（进攻型 vs 守护型）
- **核心轴 2**: mystery vs social（独行型 vs 社交型）
- **调节轴**: mind × order（策略+秩序 vs 直觉+自由）

### 4.2 16 种类型详细定义

---

#### 类型 1：守护者 / The Guardian / 守護者（しゅごしゃ）

**向量区间**: protection ≥ 7, order ≥ 6, social ≥ 5 **一句话定义**: 用责任感和行动力保护身边所有人的人

**性格描述**: 你是那种会默默站在朋友身后、在关键时刻毫不犹豫站出来的人。你不喜欢大声宣告自己的决心，但你的行动永远比语言更有力。你信奉规则和秩序，因为你相信这是保护弱者的最有效方式。你偶尔会因为承担太多而疲惫，但你从不后悔。

**光明面**: 可靠、有担当、言出必行，是所有人心中最值得信赖的存在 **阴暗面**: 容易过度承担，不愿意向别人示弱，可能会忽视自己的需求

**匹配卡牌风格**: 高 HP/DEF 的防御型卡牌、守护效果卡牌、光属性战士族

#### 类型 2：策略家 / The Strategist / 策略家（さくりゃくか）

**光明面**: 冷静、精准、在混乱中永远是最清醒的那个 **阴暗面**: 有时会过度分析导致行动迟缓，可能让人觉得冷漠

**匹配卡牌风格**: 效果复杂的策略型卡牌、陷阱卡、拥有多步连锁效果的怪兽

**光明面**: 勇敢、果断、充满行动力，是团队中最先冲锋的人 **阴暗面**: 有时候会冲动，不够重视策略和后果

## 五、匹配算法详细设计

### 5.1 算法选择：欧氏距离（Euclidean Distance）

经过对比分析，我们选择**欧氏距离**而不是余弦相似度。原因：

- 我们的向量各维度都已经归一化到 0~10，量级一致，不存在量级不均的问题
- 在人格匹配的场景下，维度的**绝对值**很重要（power=9 的人应该匹配 power=8~10 的卡牌，而不是 power=3 但方向一致的卡牌）
- 欧氏距离计算更直观、实现更简单，在 SQL 中可以直接用数学函数

### 5.2 匹配流程

```
输入：用户 6 维向量 U = [u1, u2, u3, u4, u5, u6]
      游戏源筛选 G = 'pokemon' | 'yugioh' | 'both'
      第 8 屏审美偏好 P = ['龙', '魔法'] 等

步骤 1：基础筛选
  - 如果 G ≠ 'both'，只保留对应游戏的卡牌
  - 只保留 supertype = 'Pokémon'（宝可梦）或 type 包含 'Monster'（游戏王怪兽卡）

步骤 2：计算距离
  对筛选后的每张卡 C = [c1, c2, c3, c4, c5, c6]：
  distance = sqrt((u1-c1)² + (u2-c2)² + (u3-c3)² + (u4-c4)² + (u5-c5)² + (u6-c6)²)

步骤 3：审美偏好加权
  如果用户在第 8 屏选了特定意象（如"龙"），
  对匹配的种族/类型的卡牌，distance *= 0.85（即拉近 15%）

步骤 4：取 Top-20 候选池

步骤 5：加权随机抽取
  将 Top-20 的距离转换为概率权重：
  weight_i = 1 / (distance_i + 0.1)  # 加 0.1 防止除零
  对权重归一化后，按概率随机抽取 1 张

步骤 6：确定人格类型
  根据用户向量 U 判断落入 16 种类型中的哪一种
  （按上文的向量区间条件依次匹配，如果同时满足多个类型，取匹配条件满足最多的那个）
```

### 5.3 Supabase SQL 实现

```sql
-- 计算欧氏距离并排序
SELECT 
  id, name, game, image_url,
  power, mystery, protection, mind, order_score, social,
  SQRT(
    POWER(power - $1, 2) + 
    POWER(mystery - $2, 2) + 
    POWER(protection - $3, 2) + 
    POWER(mind - $4, 2) + 
    POWER(order_score - $5, 2) + 
    POWER(social - $6, 2)
  ) AS distance
FROM cards
WHERE game = $7  -- 或 'both' 时不加此条件
  AND supertype = 'monster'  -- 只匹配怪兽卡
ORDER BY distance ASC
LIMIT 20;
```

$1~$6 是用户的 6 维向量值，$7 是游戏源。

## 七、Claude API Prompt 设计

### 7.1 批量给卡牌打标签的 Prompt

```
你是一个卡牌人格标签分析师。你需要根据卡牌的信息，为每张卡牌在以下 6 个人格维度上打分（0~10 的浮点数）：

1. power（力量倾向）: 0=温和回避 → 10=强势对抗
2. mystery（神秘感）: 0=坦率透明 → 10=深沉难懂
3. protection（守护本能）: 0=自保优先 → 10=守护他人
4. mind（思维方式）: 0=直觉感性 → 10=策略理性
5. order（秩序感）: 0=自由混沌 → 10=纪律秩序
6. social（社交能量）: 0=独行内向 → 10=群体外向

以下是经过规则层预处理后的卡牌数据。请根据卡名意象、效果描述的语义、以及整体氛围来微调每个维度的分数。微调幅度在 ±2 以内。

返回 JSON 数组格式，每个元素包含 card_id 和调整后的 6 个分数。

卡牌数据：
{cards_json}

请只返回 JSON，不要包含任何其他文字。
```

### 7.2 实时生成人格解读的 Prompt

```
你是一个温暖且有洞察力的人格解读师。根据以下用户的性格维度数据，用 {language} 写一段 3~5 句话的性格描述。

要求：
- 语气像一个了解用户很久的朋友在跟他说话
- 先说一个正面的核心特质
- 然后说一个别人可能不知道的内在面
- 最后给一句温暖的肯定
- 不要使用"你是 XX 型人格"这种标签化的表达
- 不要用"根据数据分析"这类技术性的说法

用户维度：
- 力量倾向: {power}/10
- 神秘感: {mystery}/10  
- 守护本能: {protection}/10
- 思维方式: {mind}/10
- 秩序感: {order}/10
- 社交能量: {social}/10

人格类型: {personality_type_name}
```

### 7.3 实时生成卡牌共鸣描述的 Prompt

```
你是一个卡牌命运解读师。请用 {language} 写 2~3 句话，解释为什么这张卡牌是这个用户的命运之卡。

要求：
- 把用户的性格特质和卡牌的特点联系起来
- 用"你们都是..."或"你和[卡名]一样..."的句式建立共鸣
- 让用户觉得"说的就是我"
- 保持神秘感和仪式感

用户人格类型: {personality_type_name}
用户最突出的维度: {top_dimensions}

卡牌信息:
- 名称: {card_name}
- 类型: {card_type}
- 属性: {card_attribute}
- 效果/描述: {card_description}
- 卡牌维度分数: power={card_power}, mystery={card_mystery}, protection={card_protection}, mind={card_mind}, order={card_order}, social={card_social}
```

## 八、完整数据流与技术架构

### 8.1 数据采集阶段（一次性）

```
1. 拉取宝可梦 TCG API 全量数据
   GET https://api.pokemontcg.io/v2/cards?pageSize=250&page={n}
   → 筛选 supertype = "Pokémon" 的卡牌
   → 预计 ~8,000 张

2. 拉取游戏王 YGOPRODeck API 全量数据
   GET https://db.ygoprodeck.com/api/v7/cardinfo.php
   → 筛选 type 包含 "Monster" 的卡牌
   → 预计 ~12,000 张

3. 数据清洗与统一
   → 统一 schema 写入 Supabase cards 表

4. 规则层打标签
   → 根据映射表自动计算 6 维初始分数

5. AI 精修层
   → 分批调用 Claude API（每批 15~20 张）
   → 预计 ~1,000 次 API 调用
   → 微调并更新 Supabase
```

### 8.2 用户使用阶段（实时）

```
用户打开 App
  ↓
选择游戏源（宝可梦 / 游戏王）
  ↓
选择模式（简单 / 复杂）
  ↓
答题/选标签
  ↓
前端计算 6 维向量（纯前端，无需后端）
  ↓
调用 Supabase RPC 函数（欧氏距离排序 + Top-20 + 加权随机）
  ↓
返回匹配卡牌
  ↓
[仅复杂模式] 判断人格类型（前端，条件匹配）
  ↓
[仅复杂模式] 调用 Claude API 生成解读和共鸣文案
  ↓
展示结果页
  ↓
保存到 user_results 表
  ↓
[可选] 生成分享图
```
